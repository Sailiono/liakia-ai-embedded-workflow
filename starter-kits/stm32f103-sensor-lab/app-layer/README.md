# Application Layer Contract

This directory defines the application-layer boundary for the Liakia Starter-F103 lab.

The user is responsible for:

- creating the IOC with STM32CubeMX;
- generating HAL low-level code;
- configuring USART1, I2C1, GPIO, and SWD;
- implementing the platform bridge inside the generated project.

Liakia provides:

- application entry points;
- shell command protocol;
- BMP280 sensor gate;
- telemetry frame and CRC gate;
- known-bad application cases;
- test scripts and evidence structure.

## Expected File Layout

Copy the Liakia application layer into the generated CubeMX project:

```text
Core/Inc/liakia_lab_app.h
Core/Inc/liakia_lab_platform.h
Core/Src/liakia_lab_app.c
Core/Src/liakia_lab_port_stm32f103.c   # user-side port
```

A first-pass HAL bridge template is provided at:

```text
app-layer/port-template/liakia_lab_port_stm32f103.c
```

Copy it to `Core/Src/`, then adjust `huart1`, `hi2c1`, and the LED pin if your generated project uses different names or pins.

## `main.c` Integration

```c
#include "liakia_lab_app.h"

int main(void)
{
  HAL_Init();
  SystemClock_Config();
  MX_GPIO_Init();
  MX_USART1_UART_Init();
  MX_I2C1_Init();

  LiakiaLab_Init();

  while (1)
  {
    LiakiaLab_Tick();
  }
}
```

## Platform Bridge

The generated HAL handles are bridged into Liakia through functions such as:

```c
LiakiaStatus LiakiaPlatform_I2cReadMem(uint8_t addr7, uint8_t reg, uint8_t *data, uint16_t len, uint32_t timeout_ms)
{
  return HAL_I2C_Mem_Read(&hi2c1, addr7 << 1, reg, I2C_MEMADD_SIZE_8BIT, data, len, timeout_ms) == HAL_OK
    ? LIAKIA_OK
    : LIAKIA_ERR;
}
```

## Shell Commands

First-pass command set:

| Command | Purpose |
|---|---|
| `version` | Print application, MCU, and interface identity |
| `led on/off` | Verify GPIO |
| `sensor id` | Read BMP280 chip ID |
| `sensor read` | Print BMP280 raw temperature, calibration values, and compensated value |
| `telemetry once` | Emit one telemetry frame with CRC |
| `diag i2c` | Print I2C scan summary |
| `reset` | Software reset for reset recovery gate |

`config get/set/save` belongs to the later Case C Flash persistence path. It is intentionally excluded from the base app so the first user run does not mix sensor and Flash problems.

## Known-Bad Principle

Known-bad code lives in the application layer, not in CubeMX-generated drivers. This keeps the learning boundary clear:

```text
the low-level project is generated by the user;
the issue is in application logic or integration;
Liakia uses gates and evidence to locate the problem;
after the fix, the same project can regress to PASS.
```

Start with the base `liakia_lab_app.c`, confirm shell and `sensor id`, then import a known-bad lab pack from:

```text
known-bad-cases/
```

Each case folder contains the broken file to import, a practice guide, and a separate answer key.
